OVERVIEW

What Is This?

A fully functional, production-grade ISO 8583 payment switch — the same protocol used by Visa, Mastercard, and every major card network. It accepts TCP connections from acquirer banks, routes authorization requests to issuer banks, applies real-time fraud scoring, runs end-of-day clearing and settlement, and calculates interchange and network fees per transaction.

⚡

Real-time Switch

Netty TCP server on port 8583 handles concurrent ISO 8583 connections. Sub-millisecond routing for 200+ simultaneous transactions.

🛡

Fraud Engine

5 configurable fraud rules with score-based aggregation. Decisions in <1ms, before any issuer contact.

💱

Interchange Pricing

Per-transaction interchange and network fees calculated at clearing. MCC-based rate lookup, configurable schedule, applied to net settlement.

🏦

Clearing & Settlement

Daily batch clearing at 23:00 UTC, net position settlement at 23:30 UTC. Net position includes interchange and network fee adjustments.

📊

Operations Dashboard

Browser-based admin UI with live transaction monitoring, fraud alert review queue, fee schedule management, and manual clearing controls.

☁️

AWS-Ready

Terraform IaC for ECS Fargate multi-AZ deployment. Aurora PostgreSQL, ALB + NLB, ShedLock for distributed jobs.

FRAUD DETECTION

Real-time Fraud Engine

How It Works

Every transaction is scored before it reaches the issuer. Each enabled rule contributes a weight to the aggregate score (0–100).

Score RangeActionRC
≥ 80DECLINE59 — Suspected Fraud
≥ 50FLAGContinue to issuer + alert
< 50APPROVEContinue normally

5 Rule Implementations

  • Velocity — Max N transactions per card per hour
  • Amount Limit — Flags amounts above configurable threshold ($5k default)
  • Blacklist — Instant DECLINE for blacklisted PANs (SHA-256 hashed)
  • Decline Velocity — Detects card-testing: ≥3 declines/hour
  • MCC Risk — Flags high-risk merchant categories (gambling, pawnshops, crypto)

Fraud Operations

DB-Driven Config

Rules stored in PostgreSQL with JSON parameters. Enable/disable and adjust thresholds without redeployment. Cache refreshes every 60s.

Alert Queue

Every flagged or declined transaction creates a FraudAlert record. Dashboard shows a review queue with fraud score and triggered rule names.

PAN Blacklist

PANs added as SHA-256 hashes — raw card numbers are never stored. Supports optional expiry timestamps for temporary blocks.

CLEARING & SETTLEMENT

End-of-Day Processing

Daily Schedule

Time (UTC)ProcessOutput
23:00 Clearing ClearingBatch · ClearingRecords with fees per txn
23:30 Settlement Net position per participant including fee adjustments

ShedLock (Multi-Instance)

PostgreSQL-backed distributed lock ensures only one ECS task executes clearing or settlement at a time across any number of instances.

Settlement Net Position Formula

credit = Σ amounts where participant = issuer
debit = Σ amounts where participant = acquirer

iReceived = Σ interchange received (as issuer)
iPaid = Σ interchange paid (as acquirer)
nFees = Σ network fees paid (both sides)

net = credit − debit + iReceived − iPaid − nFees
  • Issuers gain interchange income from every transaction they issue
  • Acquirers pay interchange on every transaction they process
  • Both sides pay a scheme (network) fee per transaction
  • One SettlementRecord per participant — stores full fee breakdown

Supported Controls

Manual Trigger

Dashboard "Run Clearing" button and POST /clearing/run endpoint allow out-of-schedule runs for testing.

Idempotency

Clearing is idempotent per date. Re-running for a date with a COMPLETE batch is a safe no-op.

CSV Export

GET /settlement/{date}/report and GET /api/interchange/report?format=csv for full fee breakdown.

INTERCHANGE PRICING

Interchange & Network Fee Engine

How Fees Are Calculated

fee = round(amount × bps / 10,000) + flat

Example: $100.00, MCC 5411 (retail rate)
interchange = round(10000 × 120 / 10000) + 8 = 128 ¢ ($1.28)
network fee = round(10000 × 5 / 10000) + 2 = 7 ¢ ($0.07)

Example: $100.00, MCC 7995 (high-risk)
interchange = round(10000 × 250 / 10000) + 15 = 265 ¢ ($2.65)

Rate Matching Logic

1. Look up exact MCC in interchange_rates (highest priority wins)
2. Fall back to DEFAULT catch-all rate if no exact match
3. Return 0 if no rate configured — never blocks clearing

Two Fee Categories

CategoryDirectionApplied To
INTERCHANGE Acquirer → Issuer Every cleared transaction
NETWORK Both → Network Both sides pay per transaction

MCC Persistence

DE18 (Merchant Category Code) is captured from every inbound ISO 8583 message and stored on the Transaction record. At clearing time, InterchangeService uses the MCC for rate lookup.

GAPS & ROADMAP

What's Still Missing

Core Protocol Gaps

Chargebacks & Disputes (0200 Adjustment)

No message type for post-settlement chargebacks. No dispute lifecycle or reason code mapping.

Duplicate / Retransmission Detection

No idempotency on STAN. A retransmitted 0100 with the same STAN would create a duplicate transaction.

Batch File Support (ISO 8583 / ACH)

Only real-time TCP. No file-based clearing upload or bulk authorization advices.

3D Secure / PSD2 SCA

No authentication challenge flow (DE55 / EMV data, liability shift logic).

Tiered / Qualified Interchange

Real networks apply multi-tier rates (qualified / non-qualified / rewards). Current model uses a single rate per MCC with no card-type differentiation.

Operational Gaps

Role-Based Access Control (RBAC)

Single admin credential. No read-only analyst role, no per-participant portal login, no API key management.

Audit Logging

Fraud rule changes, participant config edits, interchange rate changes, and blacklist additions are not persisted to an audit trail.

Real-time Reporting & Charts

Dashboard shows daily totals only. No hourly charts, trend lines, fraud rate over time, or fee revenue trends.

Distributed Tracing (OTEL / Zipkin)

No trace IDs propagated through multi-hop flows. Debugging a slow issuer requires log correlation by STAN.

Dead-Letter Queue & Retry

Issuer timeouts respond immediately with RC=91. No retry queue or delayed re-delivery for transient failures.
